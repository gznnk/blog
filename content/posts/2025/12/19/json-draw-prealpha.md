# AI前提で設計した、JSONで構造を持つAIネイティブな作図アプリを作りました（Pre-Alpha版）。

今年の２月にGithub Copilotを使い始めたのをきっかけに、およそ10ヶ月間、AIを活用したコーディングを続けてきました。
開発していたのは、JSONで構造を持つAIネイティブな作図アプリケーションです。
まだまだPre-Alpha版ですが、ある程度形になってきたのと、今年ももうすぐ終わるので、ここまでの振り返りを兼ねて紹介します。

## 作ったもの
まずコードがこちらです。
https://github.com/gznnk/json-draw-prealpha

そして、実際に動かせるものがこちら。
https://gznnk.github.io/json-draw-prealpha/

Pre-Alpha版なので、コードが汚かったり、UIが適当だったり、バグったりしますが、図形アプリケーションとしての基本的な機能はあらかた実装しています。

## 何を目指したのか
このアプリケーションで目指したのは、以下のような大きなポイントです。

* AIを前提とし、AIでも扱いやすいJSON形式で図形データを保存する
* AIによる補助により個人開発でどこまで大きく複雑なアプリケーションを作れるか試す

他にも
* ライブラリを極力使用せず、AIの力を借りて自前で実装することで、ライブラリの制約に縛られない設計にする
* Reactでどこまで描画性能を引き出せるか
* １枚のキャンバス上ですべての情報を管理するのは便利なのか
*そもそも作図アプリケーションってどう作るんだろうか、という技術的な興味。

などなどがあります。

## JSONはAIと相性が良かったのか？
結論、相性はいいと感じています。
yamlのほうがいいという意見も見かけますが、私が試した限りでは、JSONでも問題なく理解できていました。

例えばですが、これは実際にデモを見てもらいたいのですが、４つある画面デザインの右下のやつは、その上の図形データをもとにHTML化したやつです。各要素の位置や色が高い精度で再現されており、JSONの構造をしっかり理解していることがわかります。

また、左下にある画面デザインは、同じデザインをVSCode上でClaude Code にデザインテーマを変更してもらったやつです。これは色を変えるだけなので難易度は低いとは思いますが、JSONの加工もできることが予感されます。ここはどっちかというとデータ構造よりコンテキストウィンドウとの戦いだと思うので、LLM側に進化してもらうか、AIがJSONを分割して理解できるようにAI周辺の環境を整える必要があると思います。

他にもシークエンス図をみて、ちゃんとシークエンスも理解できますし、JSONでいいと感じています。

モデルはGPT-5を使っていて、API Keyを入れてもらえれば誰でも試せるので、興味ある方はぜひ試してみてください。
選択中の図形は何か？とか、図形の色を変えてみて？とか、図形を追加して？とか、いろいろ試せます。


## 個人開発でどこまで作れたか

Pre-Alpha版のコード量は以下の通りです。

```
Total : 1273 files, 71738 codes, 9935 comments, 7130 blanks, all 88803 lines
```

これを10ヶ月で、日中は仕事、家事や子育ての合間にコツコツと開発してきました。多分使えた時間は500時間くらいでしょうか。
一時間あたり約143行のコードを書いた計算になります。
AIが当たり前の今となっては少ない感じがしますが、初めて作る作図アプリケーションで、何度も構造を見直しながら作ってきたことを考えると、効率的に開発できたと感じています。

作ってきた機能でいうと、
* 図形の移動
* 図形の拡大縮小
* 図形の回転
* 図形の重なり順変更
* 図形のグループ化
* 図形の複製
* 図形の削除
* 図形の属性変更（色、線の太さなど）
* 複数図形の選択
* 元に戻す・やり直す
* ズームイン・ズームアウト
* 慣性スクロール
* エッジ接続
* エッジの自動ルーティング
* AI Workflow
* AIによる図形生成支援

などなど。他にも色々あると思いますが、仕事でも色々あったので、もはや何作ったのか忘れました。（火消し・トラブル対応の１年でした...）

作ってみるとわかるんですが、作図アプリケーションは１枚の画面に多くの機能を詰め込む必要があるので、まぁ難しかったです。

最初のうちは慣れない幾何学的計算をAIに教えてもらえることで、AIがない場合に比べてかなり効率的に学習しながら進められました。が、ある程度慣れて来て、シビアな状態管理やパフォーマンスチューニングが必要になると、AIはあまり役に立たなくなり、ほとんど自力で解決することが多くなりました。

この10ヶ月やってきて感じたのは、結局AI（LLM）は結局尤もらしい文章製造機でしかないので、尤もらしい文章製造機で出来そうなところ、出来なさそうなところを見極めて、適材適所で使うのが重要だな、ということです。

とは言いつつも脳の大統一理論を読んでみると人間の脳は推論してるだけという話もあるので、もしかしたらあとは推論をどううまく使いこなせるかだけなのか？とも思ったり、面白いです。

# このブログについて

このブログは **Knowledge as Code** の考え方を試すために作っています。  
知識を文章として書くだけでなく、**コードと同じように管理・運用できる形で残す**ことを目的にしています。

---

作り始めたきっかけは、**コーディングエージェントの登場**でした。  
正直なところ、こうした仕組みは以前から作ろうと思えば作れましたが、そこまでの気概はありませんでした。

AIによってコードをすぐに書ける環境が整ったことで、  
**初めて腰を上げた、というところです。**

システムエンジニアにとっての知識の資産は、**文章とコードの両方**だと考えています。

---

Zenn や Qiita はとても便利ですが、  
コードそのものを「知識として蓄積・管理する」には、少し向いていない面もあります。

このブログでは、**文章とコードを同じリポジトリで扱う**ことで、  
その点を解消できると考えています。

---

技術的には、**Markdown で記事を書き、それを HTML に変換して公開**しています。

- GitHub Actions  
- GitHub Pages  

を使って CI/CD を組んでおり、  
**Markdown を書いて push するだけで自動的に公開**されます。

ブログとしての更新フローはかなり軽く、  
思いついたことをすぐに反映することができます。

---

記事の更新履歴を **git の差分として管理できる**点も、  
この形を選んだ理由のひとつです。

どこをどう直したか、考えがどう変わったかが履歴として残るのは、  
**知識の管理方法として理にかなっている**と思っています。

---

また、**VSCode などの IDE 上で完結できる**のも大きな利点です。

普段の開発環境の延長で書けるため、  
特別なツールを意識せずに続けられます。

AIも IDE 上で使う方が作業しやすく、  
この構成は **AIとの親和性も高い**と感じています。

文章も AI と対話しながら書くことができるため、  
**知識の蓄積がよりスムーズになる**と考えています。

---

こうした背景から、**Knowledge as Code を実際に運用しながら確かめるための場**として、システムエンジニアにとって理想的な知識管理のあり方をこのブログで模索していきます。
